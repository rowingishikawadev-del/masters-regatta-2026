1

GitHub リポジトリの確認と Personal Access Token の発行

このシステムは GitHub のリポジトリにリザルトデータを書き込み、Cloudflare Pages が自動的に公開します。 まず使用するリポジトリと、書き込みに必要な認証トークンを準備します。

1-1. リポジトリの存在を確認する

ブラウザで下記の URL を開き、リポジトリが表示されることを確認してください。

https://github.com/RYUIYAMADA/rowing-live-results
ℹ
「404」が表示された場合 リポジトリが存在しないか、非公開になっています。システム管理者に連絡してください。

1-2. Personal Access Token (PAT) を発行する

GAS (Google Apps Script) が GitHub に結果を書き込むためのパスワードとして機能します。 以下の手順で発行してください。

  1. GitHub にログインした状態で、右上のアイコンをクリック → Settings を選択します。
  2. 左サイドバーを一番下までスクロールして Developer settings をクリックします。
  3. Personal access tokens → Tokens (classic) の順にクリックします。
  4. 右上の Generate new token → Generate new token (classic) をクリックします。
  5. Note 欄に用途が分かる名前を入力します（例: rowing-live-results-gas）。
  6. Expiration（有効期限）を設定します。大会期間をカバーできる期間を選択してください。
  7. Select scopes で repo にチェックを入れます。
    （repo にチェックを入れると配下の項目もすべて選択されます）
  8. ページ下部の緑色ボタン Generate token をクリックします。
  9. 表示されたトークン（ghp_... で始まる文字列）を 必ずこの画面でコピー してください。 画面を閉じると二度と確認できません。
⚠
トークンは絶対に他人に教えないでください このトークンはリポジトリへの書き込み権限を持ちます。メールや SNS で共有しないようにしてください。 漏洩した場合は即座に GitHub の同じ画面から Delete（削除）してください。

Google Drive フォルダの作成

タイマーアプリが出力する CSV ファイルをアップロードするフォルダを Google Drive に作ります。 フォルダ構成は下記のとおりに作成してください。

2-1. フォルダ構成

📁 [大会名]/
  📁 race_csv/  ← タイマーCSVをここに入れる
    📁 500m/
    📁 1000m/
  📁 master/  ← エントリーデータをここに入れる
  📁 processed/  ← 処理済みファイルが自動移動される（作成のみ）
    📁 500m/
    📁 1000m/

2-2. フォルダの作成手順

  1. Google Drive を開きます。
  2. 「マイドライブ」内の任意の場所で右クリック → 新しいフォルダ で大会名のフォルダを作ります（例: 2025年諏訪湖レガッタ）。
  3. 作成したフォルダを開き、その中に race_csv・master・processed の3つのフォルダを作ります。
  4. race_csv の中に 500m と 1000m を作ります。
  5. processed の中にも同様に 500m と 1000m を作ります。

2-3. ルートフォルダの ID を取得する

大会名のフォルダ（ルートフォルダ）を開き、ブラウザの URL バーを確認してください。 URL は次のような形式になっています。 

https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStUvWxYz

folders/ の後ろにある文字列がフォルダ ID です。 この値をコピーして、次の Step 3 で使用します。

💡
フォルダを右クリック → リンクをコピー でも同じ URL が得られます。
3

Google Apps Script の設定

Google Apps Script (GAS) は、Google のサーバー上で動く自動化スクリプトです。 Drive に CSV がアップロードされると定期的に検知し、GitHub へデータを書き込みます。

3-1. 新規プロジェクトの作成

  1. ブラウザで script.google.com を開きます。
  2. 新しいプロジェクト をクリックします。
  3. 左上の「無題のプロジェクト」をクリックして、分かりやすい名前に変更します（例: rowing-live-results）。

3-2. Code.gs の内容をコピーする

  1. エディタ画面の左側に Code.gs というファイルが表示されているはずです。クリックして選択します。
  2. エディタ内の既存テキストをすべて削除します（Ctrl+A → Delete）。
  3. このリポジトリの gas/Code.gs の内容をすべてコピーして貼り付けます。
  4. Ctrl+S（Mac は Cmd+S）で保存します。

3-3. スクリプトプロパティの設定

機密情報（トークン等）はコードに直接書かず、スクリプトプロパティとして安全に保管します。

  1. メニューバーの プロジェクトの設定（歯車アイコン）をクリックします。
  2. ページ下部の スクリプト プロパティ セクションまでスクロールします。
  3. スクリプト プロパティを追加 をクリックし、下記の3つを入力します。
プロパティ名 設定する値 取得元
DRIVE_ROOT_FOLDER_ID Step 2-3 で取得したフォルダ ID Google Drive の URL
GITHUB_TOKEN Step 1-2 で発行したトークン（ghp_...） GitHub Settings
MEASUREMENT_POINTS 500m,1000m 固定値（変更不要）
⚠
プロパティ名は半角・大文字で正確に入力してください スペルミスがあるとスクリプトが動作しません。コピー&ペーストを推奨します。

3-4. 初回の認証を行う

GAS が Drive や GitHub にアクセスするには、最初に一度だけ手動で権限を承認する必要があります。

  1. エディタ画面上部の関数セレクタ（ドロップダウン）で testRun を選択します。
  2. ▶ 実行 ボタンをクリックします。
  3. 「承認が必要です」というダイアログが表示されたら 権限を確認 をクリックします。
  4. Google アカウントを選択し、「このアプリは確認されていません」と表示されたら 詳細 → （プロジェクト名）に移動 をクリックします。
  5. 要求される権限の一覧を確認して 許可 をクリックします。
  6. 元のエディタ画面に戻り、下部の「実行ログ」に testRun completed などのメッセージが表示されれば成功です。
ℹ
「このアプリは確認されていません」の警告は、個人用スクリプトに必ず表示される Google のセキュリティ警告です。 自分自身が作成したスクリプトなので、安心して「許可」してください。

3-4b. setupAll() で一括セットアップする【推奨】

認証が完了したら、setupAll() を実行することで以下の作業を自動化できます。

  • スクリプトプロパティが正しく設定されているか確認
  • Google Drive のフォルダ構成（race_csv/, master/, processed/ および計測ポイントのサブフォルダ）を自動作成
  • GitHub API への接続テスト（テスト用ファイルを作成・削除して書き込み権限を確認）
  1. 関数セレクタで setupAll を選択して ▶ 実行 をクリックします。
  2. 実行ログに === セットアップ完了 === と表示されれば成功です。
  3. ログに [未設定] が表示された場合は、3-3 のスクリプトプロパティを再確認してください。
  4. GitHub 接続エラーが表示された場合は、ログのメッセージに従いトークンの権限またはリポジトリ名を確認してください。
💡
setupAll() を使わず手動でフォルダを作成することもできます。 Drive に以下のフォルダ構成を手動で作成してください: race_csv/500/、race_csv/1000/、master/、processed/500m/、processed/1000m/

3-5. トリガーを設定する

2分おきに自動でスクリプトを動かすためのスケジュールを設定します。 setupTrigger() を実行すると自動で設定されます。

方法A: setupTrigger() で自動設定（推奨）

  1. 関数セレクタで setupTrigger を選択して ▶ 実行 をクリックします。
  2. 実行ログに [OK] トリガーを設定しました: onTrigger (2分間隔) と表示されれば完了です。
ℹ
トリガーをリセットしたい場合は deleteTriggers() を実行すると全トリガーを削除できます。 その後 setupTrigger() で再設定してください。

方法B: 手動設定

  1. 左サイドバーの時計アイコン（トリガー）をクリックします。
  2. 右下の トリガーを追加 ボタンをクリックします。
  3. 以下のとおり設定します。
設定項目値
実行する関数onTrigger
実行するデプロイHead
イベントのソース時間主導型
時間ベースのトリガーのタイプ分ベースのタイマー
時間の間隔2分おき
  1. 設定が完了したら 保存 をクリックします。
  2. トリガー一覧に今設定した項目が表示されれば完了です。

3-6. マスターデータを投入する

  1. Drive の master/ フォルダに schedule.csv と entries.csv をアップロードします。
  2. 関数セレクタで importMasterData を選択して ▶ 実行 をクリックします。
  3. 実行ログに master.json Push完了 と表示されれば、GitHub の data/master.json が更新されています。
4

Cloudflare Pages の設定

Cloudflare Pages を使うことで、GitHub に Push されたデータが即座に Web サイトとして公開されます。 無料で使用できます。

4-1. Cloudflare アカウントの準備

cloudflare.com でアカウントを作成してください（無料）。 すでにアカウントがある場合はログインします。

4-2. Pages プロジェクトの作成（実際の画面の流れ）

  1. https://dash.cloudflare.com にアクセスします。
  2. 左サイドバー「Workers & Pages」→「Pages」→「新しいプロジェクトを作成」をクリックします。
  3. 「GitHubに接続」をクリックし、GitHub アカウントを認証します（初回のみ）。
  4. リポジトリ一覧から RYUIYAMADA/rowing-live-results を選択します。
  5. 「セットアップの開始」でビルド設定を以下のとおり入力します。

4-3. ビルド設定

このサイトはビルド不要の静的 HTML です。以下のとおり設定してください。

設定項目値
プロジェクト名任意（例: rowing-live-results）
本番ブランチmain
フレームワーク プリセットなし
ビルドコマンド（空欄のまま）
ビルド出力ディレクトリ/
ルートディレクトリ/
  1. 「保存してデプロイ」をクリックします。
  2. デプロイ完了後、表示される URL（例: rowing-live-results.pages.dev）をメモします。
  3. その URL をブラウザで開いてサイトが表示されることを確認します。
💡
カスタムドメインの設定（任意） xxx.pages.dev の URL を独自ドメイン（例: results.example-regatta.jp）に変更できます。 Cloudflare → Pages → カスタムドメイン タブから設定してください。

4-4. GitHub Actions 用シークレットの設定

サイト疎通確認ワークフローで使用する URL をシークレットとして登録します。

  1. GitHub リポジトリ → Settings → Secrets and variables → Actions を開きます。
  2. 「New repository secret」をクリックします。
  3. Name に SITE_URL、Value に Cloudflare Pages の URL（例: https://rowing-live-results.pages.dev）を入力して保存します。

4-5. 確認コマンド（ローカル）

ローカル環境から疎通確認を実行する場合は以下のコマンドを使用します。

python3 tools/check_status.py --site https://rowing-live-results.pages.dev
ℹ️
大会当日の自動確認について .github/workflows/pages-check.yml を有効にすると 10 分ごとにサイトの疎通確認が自動実行されます。 大会終了後は GitHub Actions の pages-check ワークフローを手動で disable してください（無駄なトリガー消費を防ぐため）。

5-4. ファイルのアップロードと取り込み

  1. 作成した tournament.csv・schedule.csv・entries.csv を Google Drive の [大会名]/master/ フォルダにアップロードします。

⚠
CSV は必ず UTF-8 で保存してください

また date 列と time 列は セルの書式を「文字列」に設定してから入力してください。Excelが自動変換すると取り込みエラーになります。

当日の運用手順

📋 当日作業スタッフ（学生）向けには専用マニュアルを用意しています
CSVアップロードの手順・トラブル対応・チェックリストをわかりやすくまとめています。
👉 当日スタッフ マニュアルを開く

6-1. RowingTimerWeb のファイル名設定

CSVファイル名は レース番号と計測ポイントだけで構いません。大会はフォルダで管理するため、日時の付与は不要です。 

【推奨】R{NNN}_{計測ポイント}.csv

例:
  R001_500.csv    ← Race 1, 500 地点
  R001_1000.csv   ← Race 1, 1000 地点（フィニッシュ）
  R003_500.csv    ← Race 3, 500 地点
要素説明
R{NNN}レース番号・3桁ゼロ埋め（例: R001, R012）
{計測ポイント}500m または 1000m
⚙️ 旧形式（YYYYMMDD_HHMMSS_R001_500.csv）も引き続き使用可能です。 タイマーアプリ側が自動で日時を付与する場合はそのままで問題ありません。

6-2. CSV のアップロード

  1. レース終了後、RowingTimerWeb から CSV をエクスポートします。
  2. 計測ポイントに応じて、対象フォルダにアップロードします。
    • 500m 計測 → [大会名]/race_csv/500/
    • 1000m 計測（フィニッシュ）→ [大会名]/race_csv/1000/
  3. アップロード後、約2〜3分で サイトに結果が反映されます。

6-3. 自動更新の仕組み（参考）

仕組みを理解しておくとトラブル時に役立ちます。 

1. GAS が2分ごとに起動
        ↓
2. race_csv/ フォルダの未処理 CSV を検知
        ↓
3. CSV を解析して JSON に変換
        ↓
4. GitHub の data/ フォルダに JSON を Push
        ↓
5. Cloudflare Pages が変更を検知して再デプロイ（数十秒）
        ↓
6. 観客がサイトを開くと最新の結果が表示される

6-4. トラブルシューティング

サイトが更新されない場合

  1. CSV ファイルが正しいフォルダ（500m または 1000m）にアップロードされているか確認します。
  2. ファイル名が正しいフォーマット（R001_500.csv または YYYYMMDD_HHMMSS_R001_500.csv）になっているか確認します。
  3. GAS のトリガーが有効になっているか確認します（時計アイコン → トリガー一覧）。
  4. GAS のログを確認します（下記参照）。

GAS ログの確認方法

  1. GAS エディタを開きます。
  2. 左サイドバーの 実行数（再生アイコン）をクリックします。
  3. 実行履歴の一覧が表示されます。赤いエラーアイコンがある行をクリックすると詳細が確認できます。

よくある問題と解決方法

症状原因対処方法
GAS ログに Invalid token が出る GitHub トークンが失効または誤り Step 1-2 を参照して新しいトークンを発行し、スクリプトプロパティを更新する
GAS ログに Folder not found が出る フォルダ ID が間違っている Step 2-3 を参照してフォルダ ID を再確認し、スクリプトプロパティを更新する
