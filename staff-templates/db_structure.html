エンティティ階層

YEAR 年度 （管理単位・将来的に追加予定）
year number 例: 2025 PK
label string 例: "2025年度"
TOURNAMENT 大会 data/master.json
name string 大会名  PK
dates[] string[] 開催日一覧（YYYY-MM-DD）
venue string 会場名
course_length number コース距離（m）
youtube_url string YouTube Live URL 任意
measurement_points[] string[] 計測ポイント一覧（例: ["500m","1000m"]）
SCHEDULE レーススケジュール master.json → schedule[]
race_no number レース番号 PK RowingTimerWebと一致
event_code string 種目コード（M_1X / W_2X / X_4+ 等）
event_name string 種目名（表示用）
category string M（男子）/ W（女子）/ X（混成）
age_group string 年齢区分（A〜H、J=ジュニア）任意 未使用時は列非表示
round string FA / FB / SF / H / RK / R
date string 開催日（YYYY-MM-DD）
time string 発艇時刻（HH:MM）
ENTRY エントリー（クルー） master.json → schedule[].entries[]
lane number レーン番号 PK（race_no との複合）
crew_name string クルー名 / 選手名（複数名は「A / B」形式）
affiliation string 所属（クラブ・学校名）
RESULT レース結果 data/results/race_NNN.json
race_no number レース番号 FK → SCHEDULE
updated_at ISO 8601 GAS更新日時（UTC）
lane number レーン番号 FK → ENTRY
rank number 順位（同着は同値）
finish object {time_ms, formatted} — 最終計測ポイントのタイム
split string 後半スプリット（例: "(1:50.23)"）
tie_group string 同着グループID（空文字=同着なし）
photo_flag boolean 写真判定フラグ
note string 備考（大会記録・失格等）任意
LAP 地点ごとのラップ（計測ポイント） race_NNN.json → results[].times{}
point string key 計測ポイント名 PK（例: "500m", "1000m"）
time_ms number スタートからの経過時間（ミリ秒）
formatted string 表示用タイム — M:SS.cc 形式（例: "1:48.22"）

ラップ記録CSVの仕様（RowingTimerWeb出力）

ファイル命名規則
YYYYMMDD_HHMMSS_RNNN_計測ポイント.csv
要素内容例備考
YYYYMMDD 計測日付 20260309 RowingTimerWebが自動付与
HHMMSS 計測時刻 002304 RowingTimerWebが自動付与
RNNN レース番号（3桁ゼロ埋め） R001 RowingTimerWebの race_no と一致させること
計測ポイント 地点名 500m / 1000m GASの MEASUREMENT_POINTS と一致させること
ファイル例:   20260309_002304_R001_500m.csv  /  20260309_002304_R001_1000m.csv
CSVカラム仕様
ヘッダー行（1行目）:   measurement_point, lane, lap_index, time_ms, formatted, race_no, tie_group, photo_flag, note
カラム名 型 例 用途・備考 GASでの扱い
measurement_point string 500m 計測ポイント名。ファイル名の地点名と一致 ファイル名から取得するため参照のみ
lane number 3 レーン番号。エントリーの lane と対応 結果JSONの lane に使用 必須
lap_index number 1 同一レーン内の周回インデックス（通常は1） 現在未使用（将来の多周回対応用）
time_ms number 108220 スタートからの経過時間（ミリ秒整数） 結果JSONの time_ms に使用 必須
formatted string 1:48.220 RowingTimerWebの表示用タイム（ミリ秒3桁） 無視。time_ms からセンチ秒2桁で再計算 使用せず
race_no number 1 レース番号。ファイル名の R001 と一致 ファイル名から取得するため確認のみ
tie_group string 1 or （空） 同着グループID。同着の艇に同じ文字列を入れる フィニッシュポイントの値を採用。同値→同順位判定 任意
photo_flag boolean true / （空） 写真判定フラグ（true または 1 で有効） フィニッシュポイントの値を採用 任意
note string 大会新記録 備考（失格・記録等の補足） フィニッシュポイントの値を採用 任意
GASの処理ルール
ルール内容
全ポイント揃いで処理 同一 race_no の MEASUREMENT_POINTS が全て揃った時点でJSON生成。片方のみではサイト更新されない
フォーマット統一 CSVの formatted（ミリ秒3桁）は使わず、time_ms からセンチ秒2桁（M:SS.cc）で再計算
tie_group / photo_flag 複数ポイントがある場合、フィニッシュポイント（最後の計測点）の値を採用
split 計算 フィニッシュ time_ms − 最初の計測ポイント time_ms を (M:SS.cc) 形式で格納
処理済みCSVの移動 JSON生成後、CSVを processed/500m/ / processed/1000m/ へ自動移動
アップロード先 race_csv/500m/ または race_csv/1000m/ に計測ポイントごとに格納
