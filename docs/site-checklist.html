過去の問題事例（トラブルシューティング早見表）

  • 1

    CSP（Content-Security-Policy）によるUI完全無効化

    症状: ページは表示されるがタブ・トグル・フィルター・ボタンが一切反応しない。コンソールに「Refused to execute inline event handler」エラーが出る。

    原因: _headers ファイルの CSP が script-src 'self' のみで 'unsafe-inline' が含まれていなかった。HTMLの onclick / onchange はインラインスクリプトとして扱われ全てブロックされた。

    対処: _headers の Content-Security-Policy に 'unsafe-inline' を追加してデプロイ。

  • 2

    tournament.dates が null でページ全体クラッシュ

    症状: ページが白紙になる（何も表示されない）。コンソールに TypeError が出る。

    原因: master.json の tournament.dates が null のまま本番データとして使用され、renderTournamentHeader() 内で dates.map() を呼び出した際にクラッシュした。

    対処: tournament.dates を必ず文字列の配列（例: ["2026-06-07"]）で設定する。null や空文字列は不可。

  • 3

    renderAll() と loadAll() の二重描画

    症状: タブ切替時にデータが二重に描画される・または表示が上書きされて消える。

    原因: app.js の初期化処理で renderAll() と loadAll() の両方がビューを描画するコードになっており、どちらかが古い（空の）状態で上書きしてしまっていた。

    対処: 初期化処理を1箇所に統一し、二重呼び出しを防ぐ。DOMContentLoaded イベント内での呼び出し順序を整理する。

  • 4

    ブラウザキャッシュによる古いJS読み込み

    症状: コードを修正・デプロイしたはずなのに古い挙動のまま。自分のPCでは修正済みだが別の端末や別のブラウザでは古い動作をしている。

    原因: ブラウザが app.js をキャッシュしており、更新後のファイルを取得していなかった。

    対処: index.html の <script src="js/app.js?v=YYYYMMDD"> の日付を更新してデプロイする（バージョンクエリによるキャッシュバスティング）。
