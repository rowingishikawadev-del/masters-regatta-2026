① このシステムでできること

このシステムは「計測スタッフがCSVをアップロードするだけで、レース結果が自動的に速報サイトに公開される」仕組みです。専門的なWeb知識がなくても大会当日の運用ができるよう設計されています。

🏁 速報サイトの機能

  • リアルタイム速報 — レース終了後、最大4分以内に結果がサイトに反映される計測CSVのアップロードから GAS処理（最大2分）+ Cloudflareデプロイ（約1分）
  • 種目別表示 — 男子・女子・混成、予選・決勝などでフィルタリングして表示できるフィルタバー・検索ボックスを使って絞り込み可能
  • 全レース一覧テーブル — 全レースの結果を1画面で俯瞰できるラップタイム・スプリットタイム・着順を一覧表示
  • スケジュール表示 — 発艇時刻・種目・エントリー情報をスケジュール形式で表示大会前からスケジュールのみ公開し、結果は追いかけて表示
  • 日別タブ切替 — 複数日開催の大会でも日ごとに表示を切り替えられる
  • YouTube Live 埋め込み — 大会のライブ配信URLを設定すれば速報ページ内に埋め込み表示できるmaster.json の youtube_url に設定。配信なしの場合は空欄でOK
  • スマートフォン対応 — PC・タブレット・スマートフォンいずれのブラウザでも快適に閲覧できる
  • 自動更新通知 — 新しいレース結果が追加されると画面上部にトースト通知が表示される

📋 管理者ダッシュボードの機能

  • GAS稼働状態の監視 — 自動処理が正常に動いているかをリアルタイムで確認できる最終実行から8分以内: 正常 / 15分超: 要確認（トリガー再起動が必要）
  • レース公開状況の確認 — 全レースのうち何件がサイトに公開済みかを日別タイムラインで確認できる
  • 当日チェックリスト — 前日〜競技中〜大会終了後の作業をチェックリストで管理できるチェック状態はブラウザに保存される。リセットボタンで初期化可能
  • パイプライン監視 — Drive→GAS→GitHub→Cloudflare→公開 の各ステップが正常かを視覚的に確認できる
  • テストCSVダウンロード — 疎通確認用のサンプルCSV・テンプレートをその場でダウンロードできる
  • リンク一覧 — Drive・GAS・GitHub・Cloudflareなど必要なサービスへのリンクをまとめて確認できる

⚙️ 自動化されていること（手動操作が不要なこと）

  • CSV→JSON変換 — 計測システムが出力したCSVを自動でJSON形式に変換する
  • 着順計算 — ゴールタイムから自動で着順（1位・2位…）を計算する
  • スプリットタイム計算 — 500m〜1000mのスプリットタイムを自動計算して表示する
  • GitHub Push — 変換したJSONを自動でGitHubにPushする
  • サイトデプロイ — GitHubへのPushを検知してCloudflareが自動でサイトを更新する
  • 処理済みCSV管理 — 処理が完了したCSVは自動でprocessed/フォルダに移動される

② 自動更新の仕組み

🏊
レース終了 → 計測システムがCSVを出力
RowingTimerWebが計測データをCSVファイルとして出力する
↓
📁
計測スタッフがCSVをGoogle Driveにアップロード
500mCSVは race_csv/500m/、1000mCSVは race_csv/1000m/ へ投入
500m・1000m両方が揃って初めて処理が開始される
所要時間: 計測スタッフの操作次第（通常1〜2分）
↓ 最大2分待機
⚙️
GAS（Google Apps Script）が自動検知・変換
2分間隔で自動実行。CSVを読み取り、タイム計算・着順判定を行い、JSON形式に変換してGitHubにPushする
所要時間: 約30秒〜1分
↓ 約1分
☁️
Cloudflare PagesがGitHubのPushを検知して自動デプロイ
コードが自動でビルド・公開される。操作は不要
所要時間: 約1分
↓
🌐
速報サイトにレース結果が表示される
閲覧者がブラウザをリロードすると最新結果が表示される。自動更新通知も出る
CSV投入から合計: 最大3〜4分

  • トラブル時の技術対応

  • スタッフへマニュアル送付

  • DriveにCSVをアップロード
  • 3〜4分後にサイト反映を確認

⑥ 大会前の準備手順

大会情報（スケジュール・エントリー）が確定したら、以下の手順でサイトに登録します。大会の1週間前までに完了させておくことを推奨します。
  • 1
    スケジュールCSVを作成する
    ・管理画面「④ テストCSVダウンロード」から schedule_template.csv をDL
    ・race_no（レース番号）、種目名、カテゴリ、ラウンド、日程・時刻を記入
    ・race_no はRowingTimerWebの番号と一致させること（ずれると結果が紐付かない）
    ・ファイル名は schedule.csv に変更して保存
  • 2
    エントリーCSVを作成する
    ・同様に entries_template.csv をDLして記入
    ・race_no・lane（レーン番号）・クルー名・所属を記入
    ・ペアやクルーは「選手A / 選手B」形式（スラッシュ前後に半角スペース）
    ・ファイル名は entries.csv に変更して保存
  • 3
    Google Drive の master/ フォルダに2ファイルをアップロード
    ・大会フォルダ → master/ フォルダを開く
    ・schedule.csv と entries.csv をドラッグ&ドロップまたは「+新規」でアップロード
    ・既存ファイルがある場合は上書きアップロードでOK
  • 4
    GASで importMasterData() を手動実行する
    ・script.google.com → 「マスターズ石川県大会2026」を開く
    ・関数ドロップダウンで importMasterData を選択 → 「実行」ボタンを押す
    ・ログに「完了」と出ればOK。エラーが出た場合はシステム管理者に連絡
  • 5
    速報サイトに大会情報が正しく表示されているか確認する
    ・masters-regatta-2026-3ha.pages.dev を開く
    ・大会名・日程・レース一覧・エントリー情報が正しく表示されているか確認
    ・スマートフォンでも表示を確認しておくと安心

⑦ 大会当日の操作手順

🌅 開会前（最初にやること）

  • 1
    GASトリガーを有効にする
    ・script.google.com → 「マスターズ石川県大会2026」を開く
    ・左メニュー「時計アイコン（トリガー）」→ onTrigger が2分間隔で設定されているか確認
    ・設定がない場合: 「＋トリガーを追加」→ 関数: onTrigger / 時間ベース / 2分ごと で追加
    ・一度 onTrigger を手動実行してエラーがないか確認するとさらに安心
  • 2
    管理画面でGASが「正常」になるまで待つ
    ・管理者ダッシュボード を開く
    ・「③ CSV自動アップロード監視」→ GASの欄が緑の「正常」になればOK
    ・最大8分かかる。8分以上経っても「要確認」の場合はトリガー設定を再確認
  • 3
    計測スタッフに当日マニュアルのURLを共有する
    ・LINEまたはメールで以下のURLを送付
    ・https://masters-regatta-2026-3ha.pages.dev/docs/day-manual.html
    ・スタッフがDriveにアクセスできるか事前に確認しておくと当日スムーズ

🏁 競技中（継続的にやること）

  • 管理画面を定期的に確認する — レース終了後3〜4分でサイトに反映されているか確認。「③ タイムライン」でレースが「サイト公開済み」になっているか見る特に最初の数レースは確実に確認すること。問題があれば早期に発見できる
  • GAS稼働状態を確認する — 「③ CSV自動アップロード監視」が常に緑の「正常」かを確認「要確認」になったらGASトリガーを再起動（一度削除して再設定）
  • 計測スタッフと連絡を取れる状態を維持する — CSVアップが遅れている場合の連絡手段を確保しておく

🌇 大会終了後（必ずやること）

  • 1
    全レースがサイトに公開されているか確認する
    管理画面「② レース 日別タイムライン」で全件「サイト公開済み」になっているか確認。未公開のレースがある場合はCSVが揃っているか確認
  • 2
    GASトリガーを停止する
    ・script.google.com → トリガー → onTrigger の右メニュー「削除」または「無効化」
    ・GASには90分/日の実行時間制限があるため、不要な稼働は止めること
    ・放置すると次の大会の前に制限に引っかかる可能性がある

⑧ トラブル対処法

症状まず確認すること対処法
管理画面が「要確認」になっている GASトリガーが動いているか script.google.com → トリガー → onTrigger を再設定。その後 onTrigger を手動実行して「完了」ログを確認
CSVをアップしても6分経ってもサイトが更新されない 500mと1000mの両方がアップされているか Driveの race_csv/500m/ と 1000m/ 両方にファイルがあるか確認。片方だけでは処理されない
レース番号が一致しないと言われた CSVのファイル名のレース番号とスケジュールが一致しているか ファイル名の R001 の番号がスケジュールの race_no と一致しているか確認。ずれている場合はCSVを再作成
サイトに古い結果が表示される ブラウザキャッシュ Ctrl+Shift+R（Mac: Cmd+Shift+R）で強制リロード。スマホはブラウザアプリを再起動
GASのログにエラーが出ている エラーメッセージの内容 「GITHUB_TOKEN」関係のエラーはトークン失効の可能性。GitHubで再発行してスクリプトプロパティを更新。解決しない場合はシステム管理者へ
スケジュールや選手名が間違っている schedule.csv / entries.csv の内容 CSVを修正してDriveにアップ → GASで importMasterData() を再実行 → サイトを確認
サイト自体が表示されない（エラーページ） Cloudflareのデプロイが失敗していないか dash.cloudflare.com → Workers & Pages → masters-regatta-2026 → デプロイ状況を確認。システム管理者に連絡

⑨ 現在の課題・制限事項

⚠️ 運用上の制限

  • GAS実行時間: 90分/日の上限あり — 5分×18回=90分。大会が2日間の場合も1日ごとにカウントされる。大会終了後は必ずトリガーを停止すること超過するとその日のGAS処理が止まる。大会中に止まった場合はシステム管理者へ連絡
  • CSVは500mと1000m両方が揃わないとサイトに反映されない — 1000mコースのため両ポイントの計測CSVが必要。片方だけでは保留される両方揃った時点で次のGAS実行（最大2分後）に処理される
  • 反映まで最大3〜4分かかる — GAS2分間隔+デプロイ1分のため、即時反映はできないタイムラグを関係者に事前に周知しておくと混乱が少ない
  • GitHubトークンは有効期限がある — 設定した期限によって1年以内に失効する。失効するとGASのPushが止まる年1回程度、大会前にトークンの有効期限を確認することを推奨

🔮 今後の拡張予定（検討中）

  • 年度・大会一覧ページの追加 — 複数大会の結果を年度別に一覧表示するトップページ。現在は大会ごとに個別URLでの運用2026年度以降、大会数が増えてきたタイミングで対応予定
  • 過去大会のアーカイブ機能 — 終了した大会の結果を永続的に保存・閲覧できる仕組み現状は最新大会のみ表示。過去結果参照ニーズが高まれば対応
  • LINEやメールへの結果通知 — レース結果が公開されたときに自動で通知する機能GASの拡張で対応可能。希望があれば検討
  • 県協会サイトへの組み込み — 県のボート協会サイトにサブドメインまたはiframeで組み込む県協会サイトの構築と並行して対応予定

⑩ 連絡先

  • 技術的なトラブル・設定変更 — システム構築担当者へ連絡（連絡先は別途お伝えします）
  • 大会情報の変更（スケジュール・エントリー） — schedule.csv / entries.csv を修正 → Drive にアップ → importMasterData() を実行。判断できない場合はシステム管理者へ
  • GASトークン失効・アカウント関連 — システム管理者へ。パスワード等の認証情報は電話または対面で共有
