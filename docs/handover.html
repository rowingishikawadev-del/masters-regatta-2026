③ システムの仕組み

📁
計測スタッフが CSVをGoogle Driveにアップロード
race_csv/500m/ と race_csv/1000m/ に投入。両方揃って初めて処理される
↓ 最大5分
⚙️
GAS（Google Apps Script）が自動検知・変換
5分間隔で自動実行。CSVをJSON形式に変換してGitHubにPush
↓ 約1分
☁️
Cloudflare Pagesが自動デプロイ
GitHubへのPushを検知して自動でサイトを更新
↓
🌐
速報サイトに結果が表示される
CSV投入から最大6分で公開。閲覧者はブラウザをリロードすれば確認できる

④ 大会前の準備（スケジュール・エントリー登録）

  • 1
    スケジュールCSV（schedule.csv）を作成し、Google Drive の master/ フォルダにアップロード
    テンプレートは管理画面 → ④ テストCSVダウンロード から取得。列の詳細は命名ルール文書を参照
  • 2
    エントリーCSV（entries.csv）を作成し、同じく master/ フォルダにアップロード
    クルー名・所属・レーン番号をスケジュールのrace_noと合わせて記入
  • 3
    GASで importMasterData() を手動実行
    script.google.com → プロジェクトを開く → 関数ドロップダウンで importMasterData を選択 → 実行
  • 4
    速報サイトにスケジュール・エントリーが表示されているか確認
    masters-regatta-2026.pages.dev を開いてレース一覧が表示されればOK

⑤ 大会当日の操作

  • 1
    GASトリガーを有効にする
    script.google.com → マスターズ石川県大会2026 → 左メニュー「トリガー」→ onTrigger が5分間隔で設定されているか確認。なければ追加
  • 2
    管理者ダッシュボードで「正常」を確認する
    masters-regatta-2026.pages.dev/admin/9922/ を開き、③ CSV自動アップロード監視 → 緑の「正常」になるまで待つ（最大8分）
  • 3
    計測スタッフへ当日マニュアルのURLを共有する
    masters-regatta-2026.pages.dev/docs/day-manual.html をLINE等で送付
  • 4
    競技中：管理画面でサイト反映を定期確認する
    レース終了後5〜6分でサイトに反映されているか確認。「要確認」になったらトリガーを再起動
  • 5
    全レース終了後：GASトリガーを停止する
    script.google.com → トリガー → onTrigger を削除または無効化。GASには90分/日の実行制限があるため不要な稼働は止める

⑥ よくあるトラブルと対処法

症状原因対処
管理画面が「要確認」になっている GASトリガーが止まっている script.google.com → トリガー → onTrigger を再設定して実行
CSVをアップしてもサイトが更新されない 500m・1000m両方が揃っていない もう一方のCSVもアップする。両方揃うと次のGAS実行（最大5分後）に処理される
サイトに古い結果が表示されている ブラウザキャッシュ Ctrl+Shift+R（Mac: Cmd+Shift+R）で強制リロード
GASがエラーログを出している GitHubトークンの期限切れ等 GAS → 実行ログを確認。GITHUB_TOKEN が失効している場合は GitHub で再発行してプロパティを更新
スケジュール・選手名が正しく表示されない master.json が古い schedule.csv / entries.csv を修正して Drive にアップ → GASで importMasterData() を再実行
