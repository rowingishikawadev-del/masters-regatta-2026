# ボートレース速報 ハブサイト 仕様書

**ステータス**: 未着手（マスターズ2026終了後に着手予定）
**作成日**: 2026-05-11
**対象**: 石川県ボート協会 大会速報ポータル

---

## 1. 概要

JARAの大会結果サイトと同様の2層構造。
年間大会スケジュールを一覧するハブサイトと、各大会の速報サイトで構成する。

### 全体構成

```
【層1: ハブサイト】
  rowing-ishikawa.pages.dev（仮）
  ├── 年間大会スケジュール一覧
  ├── 各大会のステータス表示
  └── 各大会サイトへのリンク

【層2: 各大会サイト】（現行システムを流用）
  masters-2026.pages.dev
  spring-2026.pages.dev
  autumn-2027.pages.dev  ...
```

---

## 2. ハブサイト機能要件

### 2-1. 大会一覧ページ（トップ）

- 年度タブ切替（例: 2026 / 2027）
- 大会カード表示（大会名・日程・会場・ステータス）
- ステータスバッジ（下記参照）
- 各大会サイトへのリンクボタン（結果なし時はグレーアウト）
- モバイル対応

### 2-2. 大会ステータス定義

| status | 表示ラベル | 説明 |
|---|---|---|
| `scheduled` | 予定 | 日程確定・まだ開催前 |
| `active` | 速報中 | 開催中・リアルタイム更新中 |
| `completed` | 結果確定 | 大会終了・結果確定済み |
| `cancelled` | 中止 | 中止・台風等 |
| `postponed` | 延期 | 日程変更あり |

### 2-3. 表示ルール

- `active` の大会は最上部に固定表示
- `scheduled` は日程順にソート
- `completed` は年度内の最下部にまとめる
- `cancelled` / `postponed` はグレー表示

### 大会の追加・変更・削除

| 操作 | 方法 |
|---|---|
| 大会追加 | tournaments.json に要素を追加 |
| 日程変更 | date_start / date_end を更新 |
| 中止 | status を `cancelled` に変更 |
| 削除 | 該当要素を削除 |

## 5. 管理パネル（Phase 2）

### 5-1. 概要

ハブサイト内の `/admin/` に配置。パスワード保護。
スタッフが tournaments.json を直接編集せずに大会を管理できる。

### 5-2. 大会一覧管理

- 大会の追加・編集・削除フォーム
- ステータス変更（ドロップダウン）
- 変更内容を GitHub API 経由で tournaments.json に保存

### 5-3. 大会ごとのコントロールパネル

各大会のカードに以下のリンク・操作を集約：

| 項目 | 内容 |
|---|---|
| 本番サイト | 速報サイトへのリンク（別タブ） |
| 大会情報の編集 | 大会名・日程・会場・エントリーの修正フォーム（GAS importMasterData をWebUIから実行） |
| 結果アップロード先 | Google Drive フォルダへのリンク（CSVを投入する場所） |
| 管理者ダッシュボード | 各大会の `/admin/9922/` へのリンク（GAS稼働状況・エラー確認） |
| ステータス変更 | scheduled / active / completed / cancelled をワンクリックで切替 |

### 5-4. 大会情報の編集フォーム（詳細）

以下の項目をWebUIから編集可能にする（現在はCSVを手動アップ）：

**大会基本情報（tournament.csv 相当）**
- 大会名
- 会場
- コース距離

**スケジュール（schedule.csv 相当）**
- レース番号・種目・カテゴリー・日時の追加・編集・削除

**エントリー（entries.csv 相当）**
- レース番号・レーン・クルー名・所属の追加・編集・削除

保存 → GAS `importMasterData()` を自動実行 → 本番サイトに即反映

## 6. 大会セットアップウィザード（Phase 2 コア機能）

### 6-1. 全体フロー

```
STEP 1: 大会登録
  大会名・日程・会場・コース距離を入力
  → 「登録」ボタン → tournaments.json に追加
        ↓
STEP 2: サイト構築
  「結果サイトを構築する」ボタン
  → GitHub リポジトリ自動作成（テンプレートから）
  → Google Drive フォルダ構造自動作成
  → Cloudflare Pages 自動登録
  → 完了画面にサイトURL表示
        ↓
STEP 3: エントリー・スケジュール登録
  「エントリー登録サイトへ」ボタン
  → スケジュール入力フォーム（種目・日時・レース番号）
  → エントリー入力フォーム（クルー名・所属・レーン）
  → 「保存して反映」ボタン → importMasterData() 自動実行
  → 本番サイトに即反映
        ↓
STEP 4: 大会当日
  Drive にCSVをアップするだけ → 自動で速報サイトに反映
```

### 6-2. STEP 1: 大会登録フォーム

| 項目 | 入力形式 | 備考 |
|---|---|---|
| 大会名 | テキスト | 例: 第18回全日本マスターズレガッタ |
| 開催開始日 | 日付 | |
| 開催終了日 | 日付 | |
| 会場 | テキスト | 例: 石川県津幡漕艇競技場 |
| コース距離 | 数値（m） | デフォルト: 1000 |
| ラップ地点 | カンマ区切りテキスト | 例: 500m,1000m（デフォルト）。1000mのみなら 1000m |
| リポジトリ名 | テキスト | 自動提案あり（例: masters-2027） |
| サイトURL名称 | テキスト | Cloudflare Pages のサブドメイン名（例: masters-2027）→ masters-2027.pages.dev になる |
| YouTube配信URL | テキスト | 任意。空欄の場合は速報サイトに表示エリアなし（詰めて表示） |

### 6-3. STEP 2: サイト自動構築

ボタン押下後に以下を順次実行（進捗バー表示）：

```
[1/3] GitHubリポジトリ作成中...   ████░░░░ 
[2/3] Driveフォルダ作成中...      ████████░
[3/3] Cloudflare Pages登録中...   ████████████ 完了
```

完了後に表示：
- 本番サイトURL
- DriveフォルダURL（当日CSV投入場所）
- 「エントリー登録サイトへ」ボタン

### 6-4. STEP 3: スケジュール・エントリー登録

**スケジュール登録（テーブル形式で入力）**

| レース番号 | 種目コード | 種目名 | カテゴリー | 年齢区分 | ラウンド | 日付 | 時刻 |
|---|---|---|---|---|---|---|---|
| 1 | M1X | 男子シングルスカル | M | G | FA | 2027-05-22 | 07:00 |
| ... | | | | | | | |

→ 行の追加・削除・並べ替えが可能

**エントリー登録（テーブル形式で入力）**

| レース番号 | レーン | クルー名 | 所属 | カテゴリー |
|---|---|---|---|---|
| 1 | 1 | 田中 太郎 | 東京ローイング | |
| ... | | | | |

→ CSVインポート機能も併設（既存CSVをドラッグ&ドロップで取込）

**保存・反映ボタン**
→ GAS `importMasterData()` を自動実行
→ 本番サイトに即時反映（〜3分）

## 7. コース・計測エッジケース対応

### 7-1. ケース①: 大会内に500mレースと1000mレースが混在

**例**: 通常は1000m大会だが、特定種目のみ500mコース

**現状の問題**:
- GASの計測ポイントは大会単位で設定（500m,1000m or 1000mのみ）
- レース単位で必要なCSVが変わると現行GASでは対応不可

**対応方針（実装時に検討）**:
- schedule.csv の `course_length` 列でレースごとにコース距離を指定
- GASが `course_length = 500` のレースは `500m` CSV のみで処理完了とみなす
- エントリー登録画面でレースごとに「コース距離」を設定できるようにする

**スケジュール登録画面への追加項目**:

| レース番号 | 種目名 | ... | コース距離 | 備考 |
|---|---|---|---|---|
| 1 | M1X | ... | 1000m（デフォルト） | |
| 5 | W1X | ... | **500m** | 500mレース |

### 7-2. ケース②: 1000mレースで500mラップが計測できなかった

**例**: 機器トラブル・計測ミスで中間タイム（500m）CSVが存在しない

**現行GASの挙動（既実装）**:
- 1000m CSV が届いていれば、500m CSV がなくてもレース結果を処理・公開する
- 速報サイトのラップ列は空欄で表示（エラーにならない）

**対応状況**:
- ✅ 結果表示: 問題なし（1000mタイムのみで順位が確定する）
- ✅ サイト反映: 問題なし
- ⚠️ 表示上の注意: ラップ欄が空欄になるため、運営側で「500m欠測」と注記できるとよい

**今後の改善案**:
- 結果入力画面から「500m欠測」フラグを手動で付与できるようにする
- 速報サイトに「※500m計測なし」のバッジを表示する

## 8. 大会自動作成機能（Phase 3）

### 6-1. 概要

管理パネルから「新大会を作成」ボタンで以下を自動実行：

1. GitHub に新リポジトリ作成（テンプレートからコピー）
2. Google Drive にフォルダ構造を作成
3. Cloudflare Pages プロジェクトを作成・GitHub連携
4. tournaments.json に新大会を追加
5. 完了後にサイトURL・GAS設定手順を表示

### 6-2. 入力項目

- 大会名
- 開催日（開始・終了）
- 会場
- リポジトリ名（自動提案あり）
- Driveフォルダ ID

### 6-3. 前提条件（事前設定）

- GitHub API Token（repo権限）
- Cloudflare API Token（Pages:Edit権限）
- GASプロジェクトの複製は手動（5分程度）
